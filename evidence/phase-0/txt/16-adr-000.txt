# ADR-000: Repository Structure

## Status

Accepted

## Context

This project is intended to simulate a senior Terraform delivery pattern, not a flat demo repository. The structure must support modular growth, environment isolation, evidence capture, and documentation continuity.

## Decision

The repository will separate:

- reusable modules under `modules/`
- environment roots under `environments/`
- architecture and delivery documentation under `docs/`
- operational helpers under `scripts/`
- auditable outputs and screenshots under `evidence/`

## Consequences

### Positive

- cleaner separation of concerns
- easier scaling across environments
- documentation remains close to delivery
- better portfolio value
- reproducible execution

### Negative

- more initial setup
- more files to maintain
- steeper learning curve than single-folder Terraform
